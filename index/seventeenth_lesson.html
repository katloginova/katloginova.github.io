Препроцессоры LESS и SASS
Препроцессор — это программа, которая берёт один тип данных и преобразует его в другой тип данных. Основная задача препроцессора — это предоставление удобных синтаксических конструкций для разработчика, чтобы упростить, и тем самым, ускорить разработку и поддержу стилей в проектах. Два найболее популярный на данный момент CSS-препроцессоров – это LESS и SCSS.
LESS имеет все базовые возможности препроцессоров и даже больше, но не имеет условных конструкций и циклов в привычном для нас понимании. Основным плюсом является его простота, практически стандартный для CSS синтаксис и возможность расширения функционала за счёт системы плагинов.
SASS (SCSS) самый мощный из CSS-препроцессоров. Имеет куда больший ассортимент возможностей в сравнении с Less. Возможности самого препроцессора расширяются за счёт многофункциональной библиотеки Compass, которая позволяет выйти за рамки CSS.

Установка пакетов LESS и SCSS

Если необходимо установить глобально, то используется команда («sudo» используется для Mac и Linux, означает от имени администратора):
npm i less -g (sudo npm i less -g)
npm i sass -g (sudo npm i scss -g)
Для того, чтобы при написании кода в LESS автоматически сразу компилировался код в CSS, необходимо установить пакет «watch»:
npm i less-watch-compiler -g
Теперь для того, чтобы запустить компилятор используется следующая команда в основной папке проекта:
less-watch-compiler less css
sass – watch scss:css
где less/scss(откуда папка), css(куда папка)
Для того, чтобы подсвечивался синтаксис кода стилей (LESS или SCSS) необходимо установить плагин LESS или SCSS.
В данных препроцессорах можно использовать переменные, миксины (функции), условия и др., более подробно об использовании и синтаксисе можно прочитать в официальной документации http://lesscss.org/ – по LESS, https://sass-lang.com/ – по SASS.
